# Mockups — generated approval-gate wireframes

Auto-generated by `research-plan` (Stage 3 Mockup+Summary-Agent), one `<slug>.html` per **UI** feature.

## What lives here

- **`<slug>.html`** — a self-contained static wireframe of the feature's planned UI: inline CSS, no build step, no external assets, no JS required to view the layout. Open it directly in a browser to preview what the feature will look like before you `/approve`.

Backend / DSP / USB / DB features have **no** file here — their mockup is a concrete text example (API response, CLI output, before→after data) embedded in the research doc's `## Mockup` section instead.

## Lifecycle

- Created when a doc reaches `approvalgate_`.
- Referenced from the doc's `## Mockup` → `### UI — mockup file`.
- Kept after merge as part of the historical record (like the research doc itself).
- Never hand-edited to change behavior — it mirrors the plan; if the plan changes, the agent regenerates it.

## Not served

These are static files for local preview only. They are **not** wired into the app, the dev server, or CI. Double-click / open in a browser.
