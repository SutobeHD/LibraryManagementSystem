5.2 Dependency Philosophy — "Schicht A"

Before listing the dependencies, it is worth understanding how the project manages them, because that policy — internally called Schicht A ("Layer A") hardening — touches every version number you will see in this chapter.

EcosystemRule
PythonEvery dependency in requirements.txt is pinned with an exact == version. Never >=, never ~=. No bump without a CVE check and a passing test run.
NodeThe lockfile is canonical. npm ci — never npm install — in CI. Install scripts are disabled.
RustCargo.lock is committed. No casual cargo update.

The reasoning is supply-chain security: an exact pin means a malicious or broken upstream release cannot silently enter a build. Chapter 21 covers the full threat model. For this chapter, the takeaway is simple — every version below is deliberate and frozen.

◆
Graceful degradation

madmom and essentia are the high-accuracy analyzers, but they are optional at import time. If they are unavailable, the analysis engine falls back to librosa. The pinned requirements.txt still lists them so a clean install produces the best analysis stack — but the application keeps working without them.

▲
pyrekordbox is the load-bearing — and fragile — dependency

The whole Live mode and the USB export depend on pyrekordbox 0.1.7. That version has known bugs: its create_content() raises on every call, and earlier versions crash the entire Python process on malformed analysis rows. The project works around both — a template-based USB writer and a subprocess-isolated parser — and Chapters 10 and 13 document the workarounds in full.

5.6 The External Dependency — FFmpeg

One dependency is not bundled or pinned: FFmpeg. It must be present on the system PATH. FFmpeg handles format conversion, sample-accurate audio slicing, render export, and the HLS-stream remux in the SoundCloud downloader.

◆
Why FFmpeg is external

FFmpeg is a large, separately-licensed native binary. Treating it as an external prerequisite keeps the installer small and the licensing clean. Every subprocess call to FFmpeg in the codebase carries an explicit timeout — a coding rule, since a hung encoder must never hang the backend.

Chapter summary

The stack is four layers: a React/Vite/Tailwind frontend; a Rust/Tauri native core built on CPAL, Symphonia and RustFFT; a Python/FastAPI backend; and a Python DSP layer built on librosa, madmom and essentia. FFmpeg is the one external prerequisite. Every dependency is exact-version-pinned under the "Schicht A" supply-chain policy, and a full four-language tooling suite gates every commit. The most powerful dependency, pyrekordbox, is also the most fragile — and the project's workarounds for it shape Chapters 10 and 13.
