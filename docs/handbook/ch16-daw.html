Chapter 16 · The Frontend

The DAW Editor

The DJ Edit DAW is a four-panel, non-destructive editing workstation built entirely in the browser — a timeline, a waveform, cue and beatgrid editing, and offline render export, all without ever rewriting a source file.

16.1 What the DAW Is For

The DAW — "DJ Edit DAW" — is where a track gets prepared. A DJ uses it to refine the beatgrid, place hot cues exactly where the breakdowns and drops are, set loops, draw volume envelopes, and trim or rearrange regions. The output is a polished, performance-ready track and a project file that records every edit.

The defining principle, repeated from Chapter 1, is non-destructive: nothing the DAW does ever modifies the original audio file. Every edit is data — a cue position, an envelope node, a region boundary — held in state and persisted to an overlay project file.

16.3 The Root Orchestrator

DjEditDaw is the root component, and it deliberately does not contain the panels' logic. Its job is orchestration:

  • It owns the DAW's editing state via a useReducer driving the composed reducer (Chapter 17).
  • It drives the playhead animation and a dead-reckoning sync that keeps the visual playhead aligned with the audio.
  • It wires the transport and editing handlers — split, ripple-delete, play, stop, jump-to, export, auto-cue.
  • It delegates everything specialised to a set of hooks.
  • It renders a single <DawLayout>, a pure presentational shell that simply places the panels.

Separating the orchestrator from the layout shell from the panels is what keeps a feature this large maintainable. DjEditDaw coordinates; DawLayout arranges; each panel renders one thing.

16.4 The DAW Hooks

The behaviour that does not belong to any single panel is extracted into custom hooks. This is the pattern the Slopcode-Cleanup used to break up the original 972-line monolithic DAW.

16.5 The Timeline Internals

The timeline is the most demanding part of the DAW — it must render a waveform, a beatgrid, regions, cues, loops, and a 60-frame-per-second playhead, all without stuttering. DawTimeline is a thin wrapper over three focused hooks under components/daw/timeline/.

HookResponsibility
useTimelineLayoutSizing — a ResizeObserver subscription, device-pixel-ratio-aware canvas sizing, debounced resize handling.
useTimelineRenderRendering — the requestAnimationFrame loop, an offscreen-canvas waveform-bitmap cache, and every draw function (grid, regions, cues, loops, playhead, ruler, phase meter).
useTimelineEventsInteraction — cue-flag hit-testing, mouse handlers for placing and dragging cues and selecting regions, and the wheel handler for scroll and cursor-anchored zoom.
◆
The bitmap cache

Re-rasterising a full waveform every frame would be ruinously slow. useTimelineRender draws the waveform once into an offscreen canvas bitmap and blits that bitmap each frame, repainting only the cheap live layers — grid, cues, the playhead — on top. The bitmap is rebuilt only when an invalidation key changes. This is what makes a 60 fps playhead affordable.

The timeline can render the waveform in several styles — a 3-band coloured view, a "liquid" view, a plain mono view, or a bass-emphasised view — chosen by a state field, with a band-specific renderer for each.

16.6 The Non-Destructive Editor

Alongside the DAW sits a second, closely related editor under components/editor/ — the non-destructive region editor. Where the DAW is oriented around preparing a track, this editor is oriented around arranging regions: cutting, inserting, deleting, and pasting slices of audio with volume envelopes.

16.7 Non-Destructive Editing in Practice

The non-destructive promise is implemented through the region abstraction. A region is not a copy of audio — it is a reference into a source file plus metadata: a start offset, an end offset, a gain, and fade curves.

An edit — splitting a region, rippling a deletion, pasting from the palette — produces a new list of regions, never a new audio file. The user hears the result because the playback engine schedules the regions in order; the user sees it because the timeline draws them. Only two operations produce a real audio file: an explicit project save (which writes a .rbep overlay) and an explicit export.

16.8 Export

Both editors can render their timeline to a real audio file through the ExportModal. The modal offers a folder picker (via the native Tauri dialog), a format choice — WAV, MP3 320, or FLAC — and a normalisation toggle.

FormatHow it is rendered
WAVRendered entirely in the browser — the Web Audio engine renders the timeline, and the result is written to disk via the native filesystem plugin.
MP3 / FLACRendered by the backend — POST /api/audio/render runs FFmpeg, and the result is downloaded and written to disk.

This split is itself a small example of the architecture's "right layer" principle: a WAV render is straightforward PCM the browser can do alone, while a compressed encode is best handed to FFmpeg in the backend. A browser fallback writes via a blob download when the native filesystem plugin is not available.

16.9 Keyboard-Driven Editing

Editing is fast because it is keyboard-driven. The DAW has fourteen rebindable shortcut actions — cut, copy, paste, delete, undo, redo, zoom in and out, scrub, jump to start and end, save, open — plus number keys 1–8 for hot-cue jumps and a Shift-held slip mode. The bindings load from settings, so a DJ can match the muscle memory of whatever they used before. The non-destructive editor has its own shortcut set (m/l/f/o/s/c/1-8/Delete/q/g) wired through useEditorKeyboard, which correctly ignores keystrokes while an input field is focused.

Chapter summary

The DAW is a four-panel non-destructive workstation orchestrated by DjEditDaw, which owns a reducer and delegates to hooks for project persistence, shortcuts, key handling, and track loading. The timeline achieves a 60 fps playhead through an offscreen-bitmap waveform cache. A companion non-destructive region editor arranges audio by reference — a region points into a source file and is never a copy — so the original recording is never touched. Both editors render to WAV, MP3, or FLAC through a shared export modal, and both are heavily keyboard-driven.
