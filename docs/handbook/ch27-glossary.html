Glossary of Terms

The vocabulary of the handbook in one place — DJ and music terms, the Pioneer format names, and the project-specific terminology.

27.1 DJ & Music Terms

TermMeaning
BPMBeats per minute — a track's tempo. The single most important number for matching two tracks to mix.
BeatgridA map of where every beat falls in a track, with the tempo of each segment — what lets hardware sync two tracks.
KeyThe musical key of a track. Two tracks in compatible keys mix harmonically.
Camelot wheelA system that maps the 24 musical keys onto a clock face (1A–12A, 1B–12B) so harmonic compatibility becomes a matter of adjacency. 8B is C major; 8A is A minor.
Harmonic mixingChoosing the next track so its key is compatible with the current one — adjacent on the Camelot wheel.
Cue pointA saved position in a track the DJ can jump to instantly.
Hot cueA cue point bound to a trigger pad — eight of them (A–H), each colour-coded.
Memory cueA saved marker for reference — typically the start of a phrase or a drop — that the DJ navigates to but does not trigger as a pad.
LoopA section of a track set to repeat.
PhraseA musical section, usually 8, 16, or 32 bars. Blending on phrase boundaries sounds clean.
DownbeatThe first, strongest beat of a bar — the anchor for the phrase grid.
Extended mixA longer version of a track (typically 5–8 minutes) with long intros and outros for blending. Contrast the Radio Edit.
TranscodeA lossy file re-encoded into a lossless container — a "FLAC" that is really an MP3 inside, betrayed by a spectral cliff.
WaveformThe visual shape of a track's amplitude over time. A 3-band waveform colours bass, mids, and highs separately.
LUFSLoudness Units Full Scale — the standard (ITU-R BS.1770 / EBU R128) measure of perceived loudness.

27.2 Hardware & Ecosystem

TermMeaning
CDJPioneer's professional DJ media player — the standard club deck. The CDJ-3000 is the current flagship.
XDJPioneer's all-in-one DJ systems — players and a mixer in one unit.
DJMPioneer's professional DJ mixer line.
RekordboxPioneer's own library-management software — the application's primary competitor.
SeratoAnother major commercial DJ library and performance application — a second competitor.

27.3 Pioneer File Formats

TermMeaning
master.dbThe SQLite database Rekordbox uses for its library — what Live mode reads and writes.
ANLZPer-track binary analysis files (.DAT, .EXT, .2EX) holding the waveform, beatgrid, and cues. See Chapter 12.
PDBThe legacy DeviceLibrary database format (export.pdb) older CDJ firmware reads from a USB stick. See Chapter 13.
OneLibraryThe newer USB database format (exportLibrary.db) that Rekordbox 6/7 auto-detects.
DeviceSQLPioneer's bespoke on-disk database engine, the basis of the PDB format.
PMAIThe four magic bytes that begin every ANLZ file.
PQTZThe ANLZ tag carrying the beatgrid.
PWAV / PWV2–PWV7The family of ANLZ tags carrying the seven waveform representations.
USBANLZThe directory on a USB stick where ANLZ files are stored, bucketed by hash.
MYSETTING.DATThe CDJ/DJM hardware-settings files written to a stick.
.rbepRekordbox Editor Project — the overlay file format the non-destructive editor saves edits to.
Crate DiggerThe community reverse-engineering effort whose documentation the PDB writer follows.

27.4 Architecture & Technical Terms

TermMeaning
SidecarThe Python backend, run as a child process the Rust shell spawns and supervises. See Chapter 4.
TauriThe Rust framework that builds the desktop window and hosts the WebView.
IPCInter-Process Communication — here, Tauri's invoke()/event bridge between the frontend and the Rust core.
WebViewThe OS-provided browser engine (WebView2 on Windows) in which the React frontend renders.
FastAPI / UvicornThe Python web framework and the ASGI server that run the backend.
CPALThe Rust crate providing cross-platform audio output.
SymphoniaThe pure-Rust audio-decoding crate.
RustFFTThe Rust crate computing the FFT for waveform analysis.
FFTFast Fourier Transform — converts a signal from the time domain to the frequency domain.
pyrekordbox / rboxThe Python library for reading and writing Rekordbox data — imported in code as rbox.
ProcessPoolExecutorThe Python mechanism that runs heavy analysis in separate worker processes.
WALWrite-Ahead Log — a SQLite journal mode; the OneLibrary writer flushes it deliberately.

27.5 Security Terms

TermMeaning
Bearer tokenThe session secret sent in the Authorization header to authenticate every mutation request.
require_sessionThe FastAPI dependency that verifies the Bearer token on a route.
PKCEProof Key for Code Exchange — the OAuth extension that secures the SoundCloud login without a stored secret.
Constant-time compareA comparison that always takes the same time regardless of the inputs, defeating timing attacks. The safe_compare helper.
CORSCross-Origin Resource Sharing — the browser policy restricting which origins may call the API.
Token bucketThe algorithm behind the rate limiter — a bucket that refills at a steady rate and empties per request.
SandboxThe validate_audio_path restriction confining file access to allowed roots.
Schicht AThe project's supply-chain hardening policy — exact pinning, disabled install scripts, signature checks.

27.6 Project-Specific Terms

TermMeaning
Standalone (XML) modeThe library mode using the application's own XML file — no Rekordbox install needed.
Live modeThe library mode operating directly on an existing Rekordbox master.db.
MelodexThe frontend's amber design system.
The research pipelineThe multi-agent feature-lifecycle system of Chapter 24.
RoutineA scheduled autonomous session that advances research-pipeline documents.
Gate (A / B / C / D)A user sign-off checkpoint in the pipeline.
SafeAnlzParserThe subprocess-isolated wrapper that quarantines the crashing pyrekordbox ANLZ parser.
Slopcode-CleanupThe six-phase code-quality remediation mission of May 2026.
rb-backendThe name of the PyInstaller-built backend executable bundled in a release.
God-componentAn over-large component (the cleanup split seven of them) — a term for technical debt, not a design.
